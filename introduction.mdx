`sem` is a semantic version control tool that shows you which **functions, classes, and methods** changed — not which lines. It sits on top of Git and uses [tree-sitter](https://tree-sitter.github.io/tree-sitter/) for accurate parsing across 21 programming languages.

```bash
sem diff

┌─ src/auth/login.ts ──────────────────────────────────
│
│  ⊕ function  validateToken          [added]
│  ∆ function  authenticateUser       [modified]
│  ⊖ function  legacyAuth             [deleted]
│
└──────────────────────────────────────────────────────

┌─ config/database.yml ─────────────────────────────────
│
│  ∆ property  production.pool_size   [modified]
│    - 5
│    + 20
│
└──────────────────────────────────────────────────────

Summary: 1 added, 1 modified, 1 deleted across 2 files
```

## The core idea

Git tracks lines. Developers think in functions. `sem` bridges the gap.

A line-level diff tells you that lines 42–67 changed. A semantic diff tells you that `authenticateUser` was modified and `legacyAuth` was deleted. The second form maps directly to how developers reason about code — and how code review, CI, and AI agents need to consume changes.

## Language support

sem parses 21 programming languages with full entity extraction via tree-sitter, including TypeScript, JavaScript, Python, Go, Rust, Java, C, C++, C#, Ruby, PHP, Swift, Elixir, Bash, HCL/Terraform, Kotlin, Fortran, Vue, XML, ERB, and Svelte.

It also understands structured data formats: JSON, YAML, TOML, CSV, and Markdown. Everything else falls back to chunk-based diffing.

See [Language support](/languages/overview) for the full breakdown of extensions and entity types per language.

## JSON output for CI and AI agents

Every command supports JSON output, making `sem` easy to integrate into automated pipelines and LLM-based tooling:

```bash
sem diff --format json
```

```json
{
  "summary": {
    "fileCount": 2,
    "added": 1,
    "modified": 1,
    "deleted": 1,
    "total": 3
  },
  "changes": [
    {
      "entityId": "src/auth.ts::function::validateToken",
      "changeType": "added",
      "entityType": "function",
      "entityName": "validateToken",
      "filePath": "src/auth.ts"
    }
  ]
}
```

Structured entity diffs are more useful to AI agents than raw line diffs because they carry semantic identity (`entityId`), change type, and file location in a stable, machine-readable format.

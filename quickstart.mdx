---
title: "Quickstart"
description: "Get entity-level diffs running in under a minute"
---

<Tip>
  No config files or setup required. `sem` works immediately in any Git repository.
</Tip>

<Steps>
  <Step title="Install sem">
    The quickest path is Homebrew:

    ```bash
    brew install sem-cli
    ```

    For other installation methods — cargo, Docker, or a prebuilt binary — see the [installation guide](/installation).
  </Step>

  <Step title="Run sem diff in any Git repo">
    Navigate to any Git repository and run:

    ```bash
    sem diff
    ```

    sem shows which entities changed in your working tree:

    ```
    ┌─ src/auth/login.ts ──────────────────────────────────
    │
    │  ⊕ function  validateToken          [added]
    │  ∆ function  authenticateUser       [modified]
    │  ⊖ function  legacyAuth             [deleted]
    │
    └──────────────────────────────────────────────────────

    ┌─ config/database.yml ─────────────────────────────────
    │
    │  ∆ property  production.pool_size   [modified]
    │    - 5
    │    + 20
    │
    └──────────────────────────────────────────────────────

    Summary: 1 added, 1 modified, 1 deleted across 2 files
    ```

    The symbols mean: `⊕` added, `∆` modified, `⊖` deleted.
  </Step>

  <Step title="Try staged changes">
    To diff only what you've staged with `git add`:

    ```bash
    sem diff --staged
    ```

    `--cached` is accepted as an alias for `--staged`, matching Git conventions.
  </Step>

  <Step title="Try JSON output">
    For use in CI pipelines or AI agents, output structured JSON:

    ```bash
    sem diff --format json
    ```

    ```json
    {
      "summary": {
        "fileCount": 2,
        "added": 1,
        "modified": 1,
        "deleted": 1,
        "total": 3
      },
      "changes": [
        {
          "entityId": "src/auth.ts::function::validateToken",
          "changeType": "added",
          "entityType": "function",
          "entityName": "validateToken",
          "filePath": "src/auth.ts"
        }
      ]
    }
    ```

    Each change carries a stable `entityId` in the form `filePath::entityType::entityName`, along with `changeType`, `entityType`, `entityName`, and `filePath` as top-level fields.
  </Step>

  <Step title="Try entity-level blame">
    See who last modified each function or class in a file — not each line:

    ```bash
    sem blame src/auth.ts
    ```

    To get blame output as JSON:

    ```bash
    sem blame src/auth.ts --json
    ```
  </Step>
</Steps>

---
title: "sem — Semantic Version Control"
description: "Entity-level diff, blame, graph, and impact analysis for code. Git tracks lines — sem tracks functions, classes, and methods across 21 languages."
---

sem extends Git with entity-level operations. Instead of seeing which lines changed, you see which functions were added, which methods were modified, and which classes were deleted. Built on tree-sitter for accurate parsing across 21 programming languages.

```bash
sem diff

┌─ src/auth/login.ts ──────────────────────────────────
│
│  ⊕ function  validateToken          [added]
│  ∆ function  authenticateUser       [modified]
│  ⊖ function  legacyAuth             [deleted]
│
└──────────────────────────────────────────────────────

Summary: 1 added, 1 modified, 1 deleted across 1 file
```

## What sem does differently

Instead of line-level noise, sem gives you entity-level signal:

<CardGroup cols={2}>
  <Card title="Semantic diff" icon="code-compare" href="/commands/diff">
    Shows which functions and classes changed — not which lines. Detects renames and moves automatically.
  </Card>
  <Card title="Entity blame" icon="user-clock" href="/commands/blame">
    Who last modified each function, not each line. Understand ownership at the semantic level.
  </Card>
  <Card title="Dependency graph" icon="diagram-project" href="/commands/graph">
    Cross-file call graph showing what each function depends on and what depends on it.
  </Card>
  <Card title="Impact analysis" icon="circle-radiation" href="/commands/impact">
    Transitive analysis: if this entity changes, what else breaks? BFS through the dependency graph.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Install sem">
    ```bash
    brew install sem-cli
    ```
    Or grab a binary from [GitHub Releases](https://github.com/yocxy2/sem/releases) or build from source with `cargo install`.
  </Step>
  <Step title="Run in any Git repo">
    ```bash
    sem diff
    ```
    No setup, no config files. Works immediately in any Git repository.
  </Step>
  <Step title="Explore other commands">
    ```bash
    sem blame src/auth.ts      # who owns each function?
    sem graph                  # dependency graph
    sem impact validateToken   # what breaks if this changes?
    ```
  </Step>
</Steps>

## Use with CI and AI agents

sem outputs JSON for easy integration with automated pipelines and AI tools:

```bash
sem diff --format json
```

```json
{
  "summary": { "fileCount": 2, "added": 1, "modified": 1, "deleted": 1, "total": 3 },
  "changes": [
    {
      "entityId": "src/auth.ts::function::validateToken",
      "changeType": "added",
      "entityType": "function",
      "entityName": "validateToken",
      "filePath": "src/auth.ts"
    }
  ]
}
```
